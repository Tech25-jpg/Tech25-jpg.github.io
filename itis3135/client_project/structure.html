The HTML Skeleton

Every HTML document starts with a basic structure, often referred to as the "HTML skeleton". This structure includes the essential elements that define the document and its content. Here's a breakdown of the key components:

  • <!DOCTYPE html>: This declaration defines the document type and version of HTML being used. It helps browsers understand how to render the page correctly.
  • <html>: The root element that wraps all the content of the HTML document. It can include attributes like lang to specify the language of the content.
  • <head>: This section contains meta-information about the document, such as its title, character encoding, and links to stylesheets or scripts.
  • <body>: This is where the visible content of the webpage goes, including text, images, links, and other media.
  • <title>: Located within the <head>, this element sets the title of the webpage, which appears in the browser tab and search engine results.
  • <meta charset="UTF-8">: This tag specifies the character encoding for the document, ensuring that text is displayed correctly across different browsers and devices.
  • <meta name="viewport" content="width=device-width, initial-scale=1.0">: This tag is crucial for responsive design, as it controls how the webpage is displayed on different screen sizes, particularly on mobile devices.

For example:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <header>
        <h1>My Website</h1>
    </header>
    <main>
        <p>Welcome to my website!</p>
    </main>
    <footer>
        <p>© 2023 My Website. All rights reserved.</p>
    </footer>
</body>
</html>

Common HTML Elements

Block-Level Elements

Tags like <div>, <h1> to <h6>, and <p> take up the full width of their parent element.

Inline Elements

Tags like <span>, <a>, and <strong> only take up as much width as necessary.

Semantic Elements

Tags like <header>, <nav>, <main>, <section>, <article>, <aside>, and <footer> provide meaning and structure to the content.

Void Elements

In HTML, Void Elements are those that do not require a closing tag, as they can close by themselves. They also cannot take on content within them.

Some common void elements include:

  • <img>: Used to embed images in a webpage. It requires the src attribute to specify the image source and the alt attribute for alternative text.
  • <br>: Represents a line break, allowing you to start a new line of text without creating a new paragraph.
  • <hr>: Creates a horizontal rule, often used to separate content sections visually.
  • <input>: Used in forms to create interactive controls like text fields, checkboxes, and buttons. It requires the type attribute to specify the type of input.
  • <meta>: Provides metadata about the HTML document, such as character encoding, viewport settings, and SEO information. It is placed within the <head> section.
  • <link>: Used to link external resources like stylesheets or prefetching resources. It also belongs in the <head>.

For example:

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Void Elements</title>
</head>
<p>First line of text</p>
<br> 
<p>Second line of text</p>
<hr>
<img src="images/coding.png" alt="Web Icon">

In the example above, the <br> tag creates a line break between the first and second lines of text. The <hr> tag adds a horizontal rule to separate content sections. The <img> tag embeds an image with the specified source and alternative text.

HTML Comments

Comments are invsible notes that you can add to your HTML code to explain what certain sections do or to leave reminders for yourself or other developers. They are not displayed in the browser and are ignored by the HTML parser.

To create a comment in HTML, you use the following syntax:

<!-- This is a comment -->

Comments can be used for various purposes, such as:

  • Explaining the purpose of a section of code.
  • Leaving notes for other developers who may work on the code in the future.
  • Temporarily disabling code without deleting it.

For different coding languages, comments may have different syntax, but the concept remains the same. Other examples of syntaxes include:

/* This is a comment in CSS */
// This is a comment in JavaScript

Using comments effectively can improve the readability and maintainability of your code, making it easier for you and others to understand the structure and functionality of your HTML documents.

Relative and Absolute Paths

When linking to resources like images, stylesheets, or other pages, you can use either relative or absolute paths.

Relative Paths are defined in relation to the current document. They are shorter and more flexible, especially when moving files around within the same project. For example, if your HTML file is in the root directory and you want to link to an image in a folder called "images", you would use a relative path like images/photo.jpg.

Absolute Paths specify the full URL to the resource, including the protocol 

http://https://github.com/
or 
https://https://github.com/
They are used when linking to external resources or when you want to ensure that the link works regardless of the location of the HTML file. For example, https://www.example.com/images/photo.jpg is an absolute path.

Using relative paths can make your project more portable, while absolute paths are necessary when linking to resources that are not part of your project. It's important to choose the appropriate type of path based on your specific needs and the structure of your project.

However, you must be careful when using absolute paths, as they can make your project less portable and harder to maintain if the domain or structure of the linked resources changes. Absolute paths are lead by the protocol: 

itis3135/client_project/css.html
or 
itis3135/components/footer.html
While relative paths are lead by forward slashes: 
/itis3135/components/header.html
or 
/itis3135/client_project/structure.html

The forward slash at the beginning of a path indicates an absolute path from the root directory of the website. However, keep in mind that you won't be able to call the root directory directly from a relative path as it will not be able to locate it.

HTML Validation

HTML validation is the process of checking your HTML code against a set of rules defined by the World Wide Web Consortium (W3C) to ensure that it is well-formed and follows best practices. Validating your HTML can help you identify and fix errors, improve accessibility, and ensure that your website works correctly across different browsers and devices.

You can use online tools like the W3C Markup Validation Service to validate your HTML code. Simply enter the URL of your webpage or upload your HTML file, and the tool will analyze your code for any errors or warnings. It will provide you with a report detailing any issues found, along with suggestions for how to fix them.

Regularly validating your HTML code is an important step in maintaining a high-quality website. It can help you catch mistakes early on, improve the user experience, and ensure that your site is accessible to all users.
