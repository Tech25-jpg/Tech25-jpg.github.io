Project Overview: Web Dev Quest

Project Description

Description: The content of this page will be an interactive web application designed to teach users the fundamentals of web development through an educational experience. The project aims to provide an engaging and educational platform for beginners to learn HTML, CSS, and JavaScript while completing quests and challenges.

Intended Users: University students and hobbyists with no prior programming experience who require a structured, visual introduction to web development.

Proposed Pages:

  • Home Page (index.html): The brief overview on what the site is about and its learning objectives.
  • Structure Page (structure.html): Details about the HTML structure and layout.
  • Style Page (css.html): Information on CSS styling and design.
  • Logic Page (js.html): An introduction to JavaScript, focusing on basic programming concepts, DOM manipulation, and event handling.
  • Quiz Page (quiz.html): Interactive quizzes that test users' understanding of the material covered in the previous pages, with real-time feedback and explanations for correct and incorrect answers.

Page Specifications

1. Home Page (index.html)

  • Purpose: To provide a welcoming introduction to the Web Dev Quest, outlining the learning objectives and providing navigation to the other pages.
  • Audience: University students and hobbyists with no prior programming experience who are interested in learning web development.
  • Content: A brief overview of the site's purpose and learning objectives.
  • Data Entry: No.
  • Validations: None.
  • Interactivity: The page will include interactive elements such as navigation links and a simple contact form. (AJAX)
  • Actions: Clicking the "Start Quest" button will navigate the user to the first lesson.

2. Structure Page (structure.html)

  • Purpose: To teach the fundamentals of HTML structure and layout.
  • Audience: University students, hobbyists, and beginners.
  • Content: Explanations regarding semantic markup, head/body relationship, and nesting.
  • Data Entry: No.
  • Validations: None.
  • Interactivity: jQuery-UI Accordion used to toggle between different tag categories.
  • Actions: Clicking headers expand lesson details to avoid information overload.

3. Style Page (css.html)

  • Purpose: To teach the fundamentals of CSS styling and layout.
  • Audience: University students, hobbyists, and beginners.
  • Content: Explanations regarding selectors, properties, and layout techniques.
  • Data Entry: No.
  • Validations: None.
  • Interactivity: jQuery-UI Accordion layout and typography.
  • Actions: Clicking headers expand lesson details about CSS coding snippets and (or) visual examples.

4. Logic Page (js.html)

  • Purpose: To teach the fundamentals of JavaScript logic and programming.
  • Audience: Students moving forward in their programming journey.
  • Content: Explanations regarding variables, functions, control structures, and DOM manipulation.
  • Data Entry: Yes.
  • Validations: Yes. Real-time validation for user inputs.
  • Interactivity: Custom JavaScript functions used to trigger events.
  • Actions: Clicking a "Run Logic" button processes the input and provides immediate feedback on certain fields.

5. Quiz Page (quiz.html)

  • Purpose: To assess the understanding of HTML structure and layout.
  • Audience: University students, hobbyists, and beginners who've completed the lessons.
  • Content: Questions testing knowledge of semantic markup, head/body relationship, and nesting.
  • Data Entry: Yes. Radio buttons, textboxes, and dropdowns for answering questions.
  • Validations: Yes. Strict form validation.
  • Interactivity: Submit Results button that triggers a script to calculate the final score.
  • Actions: On click, the data is validated, score is computed, and a summary is displayed via dynamic div.

Dynamic Functionality

  • jQuery-UI Accordion: An interactive accordion component for displaying content in a collapsible manner, while also preventing an influx of information overload.
  • AJAX: Asynchronous JavaScript and XML for updating parts of a web page without reloading the entire page.
  • Form Validation: Real-time validation for user inputs to ensure data integrity and improve user experience. Both the logic and quiz pages will have these while using the submit button event, and updating a div with results. The validation will also ensure that the user didn't leave their answer blank.
